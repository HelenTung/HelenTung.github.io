准备与本机环境

在开始本文之前，应该首先阅读efb-qq-slave作者写的教程安装并使用EFB在Telegram收发QQ消息。熟悉一些基本的概念。

环境搭建

申请一个telegram Bot 用于转发wechat消息

要创建一个新的Bot，要先向@BotFather发起会话。发送指令/newbot以启动向导。期间，你需要指定这个 Bot的名称与用户名（用户名必须以bot结尾）。完毕之后@BotFather会提供给你一个密钥（Token），妥善保存这个密钥。请注意，为保护您的隐私及信息安全，请不要向任何人提供你的Bot用户名及密钥，这可能导致聊天信息泄露等各种风险。

接下来还要对刚刚启用的 Bot 进行进一步的配置：允许 Bot 读取非指令信息、允许将 Bot 添加进群组、以及提供指令列表。

  • 发送 /setprivacy 到 @BotFather，选择刚刚创建好的Bot用户名，然后选择 “Disable”.
  • 发送 /setjoingroups 到 @BotFather，选择刚刚创建好的Bot用户名，然后选择“Enable”.
  • 发送 /setcommands 到 @BotFather，选择刚刚创建好的Bot用户名，然后发送如下内容：
help - Show commands list.
link - Link a remote chat to a group.
unlink_all - Unlink all remote chats from a group.
info - Display information of the current Telegram chat.
chat - Generate a chat head.
extra - Access additional features from Slave Channels.
update_info - Update info of linked Telegram group.
react - Send a reaction to a message, or show a list of reactors.
rm - Remove a message from its remote chat.

获取telegram ID

通过 Bot 查询，很多现存的Bot也提供了ID查询服务，直接向其发送特定的指令即可获得自己的数字ID。

robot指令
  • @get_id_bot 发送 /start
  • @XYMbot 发送 /whois
  • @mokubot 发送 /whoami
  • @GroupButler_Bot 发送 /id
  • @jackbot 发送 /me
  • @userinfobot 发送任意文字
  • @orzdigbot 发送 /user

基础应用安装

在你的服务器上输入命令安装以下应用

yay -S libopus0 ffmpeg libmagic1 git libssl-dev git

enForwarderBot

pip3 install ehforwarderbot

参考官方安装方式、此处是ehForwarderBot的github网址。

efb-telegram-master

pip3 install efb-telegram-master

参考官方安装方式、此处是efb-telegram-master的github网址。

efb-wechat-slave

pip3 install -U git+https://github.com/ehForwarderBot/efb-wechat-slave

此处不能参考官方安装方式、否则会出现以下报错：

xml.parsers.expat.ExpatError: mismatched tag: line 64, column 4，~/.ehforwarderbot/profiles/default/blueset.wechat 不存在wxpy.pkl文件 ##142

若出现xml.parsers.expat.ExpatError:报错，详情请见github_issues

config.yaml

cd ~/.ehforwarderbot/profiles
mkdir wechat
mkdir blueset.telegram
mkdir blueset.wechat
touch config.yaml
## 新建blueset.telegram之后进入
cd blueset.telegram
touch config.yaml

执行以下bash脚本

WECHAT_WEB_CONFIG=~/.ehforwarderbot/profiles/wechat
TELEGRAM_MASTER="blueset.telegram"
WECHAT_SLAVE="blueset.wechat"
configure_wechat_web() {
  (
    echo "master_channel: ${TELEGRAM_MASTER}"
    echo "slave_channels:"
    echo "  - ${WECHAT_SLAVE}"
  ) >> "${WECHAT_WEB_CONFIG}/config.yaml"

  (
    echo 'token: "<your_token>"'
    echo "admins:"
    echo "  - <your_id>"
  ) >> "${WECHAT_WEB_CONFIG}/${TELEGRAM_MASTER}/config.yaml"
}

其中your_token填入你申请的 Bot 密钥、your_id 填入你的 telegram ID。

启动服务

暂时启动服务可以使用此种方式，但会随着会话终端的结束而结束

ehforwarderbot --profile wechat

可以参考以下做法，编写efb.service，使用systemctl托管服务。efb.service文件内容如下：

[Unit]
Description=ehforwarderbot
After=network.target

[Service]
ExecStart=/usr/local/bin/ehforwarderbot --profile wechat 

[Install]
WantedBy=default.target

或者使用nohup命令、将结果定向至code.out文件、扫码登陆即可。

nohup ehforwarderbot --profile default > code.out & 

cat code.out

此时，你的 wechat Bot就可以转发微信消息了，如果需要指定联系人，需要新建 telegram 群组、并且添加自己创建的 Bot 为管理员、使用 Bot 指令的/link xxx（联系人微信名称或备注），注意，在微信联系人中不能存在同个微信昵称的联系人，否则就无法绑定、也无法解绑，详情请见此意见。

robot详解
  1. /help 列出所有指令的作用
  2. /link 绑定联系人到群组
  3. /unlink_all 解绑群组的联系人
  4. /info 获取联系人的绑定号（应为具体的负数）
  5. /chat 指定某个联系人发起会话
  6. /extra 其他额外的功能
  7. /update_info 更新同步联系人头像和昵称到群组
  8. /react 选中信息并且回复联系人
  9. /rm 选中信息撤回
